Hexo和GitHub快速搭建个人博客

前言

  1. 最近不是很忙，闲下来整理一些自己CSDN和OneNote上记录的技术点，还有一些技术书籍的笔记，CSDN总感觉不是很方便，习惯性每次都是记到OneNote上，然后再发到CSDN，排版很别扭。后来打算在自己的云服务器上写一个博客网站，但是确实费时费力估计自己设计的界面也不会太好看，偶然听朋友说hexo + github可以轻松的搭个博客，编辑可以用mark down，也有很多主题。今天第一篇记录搭建的过程，和网上其他教程大同小异，后续会把之前的技术积累陆续转移到这个博客上。

准备

  1. GitHub账号一个(GitHub相关配置等不作说明)。
  2. 安装node.js（官网下载）。
  3. 安装git（mac自带）。
  4. GitHub创建username.github.io的目录（格式固定，以后也是访问的url地址）。

开始

安装hexo

  1. 因为装了node，所以有了npm命令，安装hexo。
    npm install -g hexo
  2. 本地创建、进入hexo目录。
    hexo init
    npm install
  3. 这时候hexo就算安装好了。

测试hexo是否安装成功

  1. 生成html文件。
    hexo g
  2. 测试运行。
    hexo s
  3. 打开localhost:4000即可看见默认首页。

绑定GitHub

  1. 修改_config.yml，因为yml格式缘故“:”后需留一个空格！
     deploy: 
     type: git
     repo: git@github.com:username/username.github.io.git
     beanch: master
  2. 安装git插件。
    npm install hexo-deployer-git --save
  3. 安装过程中会提示输入git相关信息，输入即可。
  4. 清除缓存、生成页面、部署(后续更新博客和主题也是这三个命令)。
     hexo clean
 hexo g
 hexo deploy
  5. 输入URL，即GitHub目录地址就可以访问了。

更换主题

  1. 官方主题上有很多主题可供选择，点击进入对应主题GitHub主页，clone到本地themes目录下即可，例如themes/iLiKE(本主题)。
  2. 修改_config.yml的theme为主题名例如：
    theme: iLiKE
  3. _config.yml也提供了其他属性的配置，例如网站标题，显示语言等，官网有具体说明可供修改。
  4. 执行更新部署的三个命令再次访问即可。
  5. PS：我在部署时候遇到了Chrome的缓存问题，导致页面加载有问题，清除缓存即可。

更新文章

  1. hexo根目录执行hexo new ‘文章名’，source->_posts即出现文件，MarkDown编辑器编辑即可。
  2. 更新执行上述三个命令即可

绑定域名

  1. 暂时没有绑定域名的需要，绑定流程也不是很麻烦，这里因为没有操作过，所以就不详细描述了。

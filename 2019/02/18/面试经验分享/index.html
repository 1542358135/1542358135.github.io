面试经验分享

前言

  1. 闲聊：
    • 又到了一年之初，很多朋友又陆续开始面试找工作了，这篇博客本来4个月前就简单写好了，一直没整理就放在电脑里没发，最近正好整理一下。这次面试先后面试了不到十家公司，发offer的概率还是比较高的，不管有没有参考价值简单总结一下吧。
  2. 根据厂规模大致分类：
    • 主要记录开发方面，理想抱负什么的就不聊了。技术方面中小型公司和更大一点的公司还是有区别的，中小型公司一般问Java基础、框架中间件（mybatis、spring、Redis、MQ、数据库、微服务等等）比较多，原理大方向上描述清楚就可以了，一般是两面技术（可能二面领导）+ 最后人事，一共三面。大公司偏向基础（基础算法、基础数据结构、jvm、Java基础rt包、并发、设计模式、网络基础、mysql原理或出题等等），相对中小公司问的比较细、杂，涉及的方面多，而且有一些需要手写代码，当时去某厂面试的时候让手写工厂、单例模式，二分查找、冒泡，面试官出题手写等等，一般三、四面技术（其中后面是领导），最后人事，正常每面都在一小时或更长时间，所以从上午面到下午的事很正常，有一次就是这种情况，在下午第一场（第三面技术）时，困得要命眼神都迷离了，表现的很差，所以保持状态也是很重要的。
  3. 关键的问题：
    • 保持一个良好的聊天环境。我不善于沟通，吃了很大的亏，后续一定要多培养。
    • 和面试官能聊的来。人家问10个问题，你8个说不上，2个回答的也不好，不用多说自己也能感觉出来。如果聊的来，很多时候都是顺着一个问题延伸下去，一两个小时问3、4个问题就过去了，有一种知己难求、接上暗号的感觉，就差不多了。
    • 最重要的是自信，自信是拿书、源码、经验堆起来的，当然既然是面试，多少也要刷刷面试题，毕竟面试东西多、杂，一般工作单一，很多问题不一定会遇到，如果有时间还是要系统的整理一遍，毕竟只看面试题深入问问就露馅了，主要是平时多研究，没什么好办法。

遇到的面试题

  1. 自我介绍：
    • 简单准备一下即可
  2. 聊聊过往项目吧：
    • 这个很重要，按着简历的项目顺序介绍，一是项目是干什么的，二是你干了什么，三是项目（你写的部分）效果怎么样，其他自行补充，不要太长，能表达出来即可，说的过程中如果面试官感兴趣会提问，如果不感兴趣说多了也不会有什么好印象，我当时没准备充分，吃过亏。
  3. 并发问题：
    • 一定会问的问题，并发问题怎么产生的，简单举例各线程私有副本和主存之间产生传递赋值问题的场景，然后就会聊到解决，一般就是锁了，锁的一些分类公平非公平，可重入不可重入，乐观悲观…，锁的三个特性，原子、可见、有序简单描述，顺着这个问题下面就是volatile和synchronize区别，atomic的场景，threadlocal，Hashmap能安全吗，ConcurrentHashmap1.8中锁怎么用的，和Hashmap区别…无穷无尽的combo

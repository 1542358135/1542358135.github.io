Java并发概念和使用

Java线程基本概念

  1. 进程和线程的关系：
    • 简单理解，一个进程可以有1个或多个线程，单个线程内顺序执行，多线程同时刻可认为是并发执行。
  2. Java中的线程使用：
    • 实现方式：
      • 继承thread类
      • 实现runnable接口
    • PS：线程start()后，执行顺序不一定按照调用顺序执行。
  3. 线程的基本使用：
    • currentThread()：获取执行当前代码线程的信息
    • isAlive()：判断当前线程是否在执行中
    • sleep()：暂停线程执行，这个方法经常和Object类的wait()方法对比，他们都有停止线程执行的效果，后面写。
    • interrupt()：中断线程，不一定能立即中断线程，只是标记为中断状态为true，等到一个可以中断的时机，这个时机比如循环中手动做一个逻辑判断，跳出循环；sleep时中断，会爆出中断异常等。
    • interrupted()：返回中断标记，并复原标记状态。
    • isInterrupt()：返回中断标记。
    • stop()：暴力终止（Java已经建议停用了）
    • yield()：让出CPU，执行其他任务，随机时间后继续执行，感觉就是一个随机暂停时间的sleep()。
    • setPriority()：设置执行的优先级，1-10越高越有可能先执行，但也不是绝对的，只能保证大部分优先级高的先于优先级低的。

Java内存模型（JMM）和线程

  1. 线程安全：
    • 什么是线程安全：Java虚拟机13章中的一句话”当多个线程访问一个对象时，如果不用考虑这些线程在运行时环境下的调度和交替执行，也不需要额外的同步，或者在调用方任何其他的协调操作，调用这个对象的行为都可以获取正确的结果，那这个对象时线程安全的”。
  2. Java内存模型和线程：
    • PS：这个问题在深入理解Java虚拟机第12 13章有详细说明，涉及到了一些硬件不是很熟，这里只带过一些并发中涉及的概念。
    • 硬件的效率优化：
      • CPU和高速缓存：由于CPU和存储设备的处理速度差距较大，加入了高速缓存作为两者间的缓冲，问题是每个CPU有自己的高速缓存，但是内存又是共享的，就出现了副本不一致，用现对应的读取协议解决。
      • 指令重排：为了使处理器内部的运算单元会尽可能充分利用，对处理的指令可能被乱序处理，只会保证结果一致。所以如果存在计算任务间的依赖，代码并不能保证顺序，这也是保证并发问题三种特性中的顺序性问题的根源。
    • Java内存模型（JMM）：
      • 主内存（Main Memory）：所有的变量都存在主内存，可以理解为堆的对象实例数据，但并不一样。
      • 工作内存（Working Memory）：每个线程会有自己的工作内存，Working Memory是高速缓存和寄存器的抽象。工作内存保存了对应线程使用主存变量的副本，处理后保存回主存。
    • 内存间交互操作：
      • 简单说分为对主存数据的操作和工作内存数据的操作，主存数据可锁定（lock）、解锁（unlock）、读取到工作内存、放回主存；工作内存可载入、使用、赋值、存储。
    • 原子性、可见性、有序性
      • 原子性：内存间交互一般可以保证基本数据类型的访问读写都是原子性的，但很多情况需要更大范围的原子性保证（例如常见的多步骤操作），这时用到了命令中的lock和unlock，Java中提供了monitor的概念实现lock和unlock，代码中常用的就是synchronize
      • 可见性：共享变量被写回主存时，在其他工作内存如果有副本（如果没有拿到是最新的），就会发出信号通知其他工作内存将该变量的缓存行设置为无效状态。当其他工作内存使用这个变量时，发现这个变量无效时，会从新从内存中读取这个变量Java中synchronize、volatile可实现。
      • 有序性：防止指令重排导致的问题（volatile、synchronize）
